2. HP E SP — CÁLCULO BASE
HP
HP_máximo = HP_base_da_classe × (1 + VIT × 0,01)

HP_base_da_classe é um valor fixo por classe que cresce por nível (definido na tabela de classe).
VIT aplica multiplicador percentual sobre esse base.
Regeneração natural começa em 0/segundo e acumula +1/s a cada 5 VIT.
Cada 200 HP máximo adiciona +1/s de regeneração natural.

SP
SP_máximo = SP_base_da_classe × (1 + INT × 0,01)

Regeneração natural de SP: +1/s a cada 2 INT e +1/s a cada 6 INT (acumulam).
Na prática: a cada 2 INT ganha +1/s, a cada 6 INT ganha mais +1/s adicional.


3. ATAQUE FÍSICO — CÁLCULO
O ATQ físico total é a soma de três componentes:
ATQ_total = ATQ_de_atributos + ATQ_da_arma + ATQ_de_equipamentos
ATQ de Atributos (Status ATK)
Vem de FOR, DES e SOR conforme regras da seção 1. É sempre neutro (sem propriedade elemental por padrão).
ATQ da Arma (Weapon ATK)
Valor fixo da arma. É afetado por refinamento e tem propriedade elemental própria.
ATQ de Equipamentos (Equip ATK)
Bônus somatórios de equipamentos e cartas que não sejam a arma.

Fórmula de dano físico por ataque normal:
Dano = ATQ_de_atributos + ATQ_da_arma × modificador_raça_tamanho + ATQ_de_equipamentos × modificador_raça_tamanho
Importante: modificadores de raça/tamanho (ex: +20% contra humanoide) se aplicam APENAS sobre ATQ_da_arma + ATQ_de_equipamentos, nunca sobre ATQ_de_atributos.
Exemplo prático:

ATQ_atributos = 71, ATQ_arma = 120
Sem modificador de raça: dano = 71 + 120 = 191
Com +20% contra humanoide (carta): dano = 71 + (120 × 1,2) = 71 + 144 = 215

Fórmula de dano por habilidade física:
Dano_habilidade = (ATQ_atributos + ATQ_arma + ATQ_equip) × multiplicador_da_habilidade

4. ATAQUE MÁGICO — CÁLCULO
ATQM_total = ATQM_base + ATQM_da_arma + ATQM_de_equipamentos

ATQM_base: vem de INT, DES, SOR conforme seção 1
ATQM_da_arma: valor da arma equipada (mais impactante dos três)
ATQM_de_equipamentos: bônus de equipamentos/cartas

Ataques mágicos são sempre afetados pela propriedade elemental do ataque vs. resistência do alvo.

5. VELOCIDADE DE ATAQUE (ASPD)
Fórmula:
ASPD = (Classe_base + Arma_base + Escudo_base) + (√(AGI² ÷ 2 + DES² ÷ 5) ÷ 4) + ((AGI + STD) ÷ 200)

Classe_base: valor fixo por classe (ex: guerreiro = 156, arqueiro = 152, etc.)
Arma_base: valor negativo ou positivo dependendo do tipo de arma (espadas leves penalizam menos; armas grandes penalizam mais)
STD: bônus direto de habilidades específicas
Limite máximo: 190 (nível 1–99)

Conversão ASPD → ataques por segundo:
Ataques_por_segundo = 50 ÷ (200 - ASPD_final)
ASPDAtaques/segundo1501,01752,01843,11884,21905,0
Atenção para implementação: bônus de ASPD em % NÃO somam sobre o valor exibido. Somam sobre o valor real com casas decimais. Um personagem com ASPD real 180,06 que recebe +5% fica com 180,81 — que ainda aparece como "180" na UI.

6. PRECISÃO E ESQUIVA — SISTEMA DE ACERTO/ERRO
Este é o coração do sistema de combate. Toda vez que um personagem ataca outro:
Passo 1 — Checar Esquiva Perfeita do defensor
Rola chance_esquiva_perfeita_defensor %
Se sucesso → ataque ERRA completamente (exibe "Lucky")
Se falha → vai para Passo 2

Esquiva Perfeita base = 0 (personagens começam com 0)
+1 Esquiva Perfeita a cada 10 SOR
Equipamentos podem adicionar mais

Passo 2 — Calcular chance de acerto normal
Chance_de_acerto = Precisão_atacante - Esquiva_defensor
Se resultado ≥ 100: acerto garantido (100%)
Se resultado ≤ 0: mínimo de 5% de chance de acerto (nunca 0%)
Rola número aleatório 1–100:

Se número ≤ Chance_de_acerto → ACERTO
Se número > Chance_de_acerto → ERRO

De onde vêm Precisão e Esquiva:
Precisão:
Precisão_total = Precisão_base_da_classe + (DES × 1) + (SOR ÷ 3) + bônus_de_equipamentos
Esquiva:
Esquiva_total = (AGI × 1) + (SOR ÷ 5) + bônus_de_equipamentos
Caso especial — ataques de habilidade:
Habilidades físicas que garantem acerto (ex: habilidades com "ignora esquiva") pulam os Passos 1 e 2 completamente.

7. DEFESA — REDUÇÃO DE DANO
Quando um ataque físico acerta, o dano é reduzido pela DEF em dois estágios:
DEF Leve (Soft DEF) — subtração direta:
DEF_leve = (VIT × 0,5) + (AGI × 0,2) + bônus_de_habilidades_defensivas
Dano_após_soft = Dano_bruto - DEF_leve
DEF Pesada (Hard DEF) — redução percentual:
Redução% = DEF_pesada ÷ (DEF_pesada + 500) × 100
Dano_final = Dano_após_soft × (1 - Redução% ÷ 100)
Exemplo: 500 DEF pesada → redução de 50%. 1000 DEF pesada → redução de ~66%.
Para dano mágico — DEFM:
DEFM Leve (Soft MDEF) — subtração:
DEFM_leve = (INT × 0,5) + (DES × 0,1) + (VIT × 0,1)
Dano_mágico_após_soft = Dano_mágico_bruto - DEFM_leve
DEFM Pesada (Hard MDEF) — redução percentual:
Redução_mágica% = DEFM_pesada ÷ (DEFM_pesada + 125) × 100
Dano_mágico_final = Dano_mágico_após_soft × (1 - Redução_mágica% ÷ 100)
Exemplo: 125 DEFM pesada → redução de 50%.

8. CRÍTICO
Chance de crítico:
CRIT = (SOR × 0,3) + bônus_de_equipamentos
Quando um ataque físico vai ocorrer (após passar o teste de acerto), rola chance de crítico:
Rola 1–100
Se número ≤ CRIT% → CRÍTICO
Efeitos do crítico:

Dano +40% sobre o dano físico final
Ignora a esquiva normal (mas não ignora Esquiva Perfeita)
Exibe balão vermelho na UI

Resistência a crítico do defensor:
Redução_de_crítico_recebido = SOR_defensor ÷ 5 %
Dano final com crítico:
Dano_crítico = Dano_físico_normal × 1,4 × (1 - resistência_crítico_defensor ÷ 100)

9. CONJURAÇÃO — TEMPO PARA HABILIDADES
Cada habilidade tem dois tempos: conjuração variável (pode ser reduzida) e conjuração fixa (não pode ser reduzida por atributos).
Fórmula completa:
χ = DES × 2 + INT
Redução_variável_por_atributos = √(χ ÷ 530)

Conjuração_variável_final = Tempo_base_variável × (1 - Redução_variável_por_atributos) × (1 - Bônus_equip_variável ÷ 100)

Conjuração_fixa_final = (Tempo_base_fixo - Redução_fixa_em_segundos) × (1 - Maior_bônus_fixo_equip ÷ 100)

Conjuração_total = Conjuração_variável_final + Conjuração_fixa_final
Regra prática para o agente de código:

Quanto mais DES + INT o personagem tem, mais rápido ele conjura habilidades variáveis
Quando χ = 530: redução de ~100% da parte variável (conjuração variável vai a 0)
Conjuração fixa nunca vai a 0 por atributos — só por equipamentos específicos


10. PÓS-CONJURAÇÃO (DELAY)
Após usar uma habilidade, o personagem fica bloqueado por um tempo de pós-conjuração. Durante esse tempo, não pode usar nenhuma habilidade (mas pode se mover e atacar normalmente).
Pós_conjuração_final = Tempo_base_delay × (1 - Soma_reduções_delay ÷ 100)
Importante — não confundir com Recarga (Cooldown):

Pós-conjuração: bloqueia TODAS as habilidades temporariamente
Recarga: bloqueia apenas AQUELA habilidade específica, as outras ficam livres


11. REGENERAÇÃO — HP E SP
Regeneração ocorre a cada segundo fora de combate ativo (ou continuamente se o personagem tiver itens/habilidades que permitam dentro de combate):
Regen_HP_por_segundo = Regen_base_VIT + (HP_máximo ÷ 200)
Regen_SP_por_segundo = Regen_base_INT
Onde:

Regen_base_VIT = (VIT ÷ 5) arredondado para baixo
Regen_base_INT = (INT ÷ 2) + (INT ÷ 6) arredondados separadamente e somados


12. SEQUÊNCIA COMPLETA DE RESOLUÇÃO DE COMBATE
Cada vez que o Personagem A ataca o Personagem B, executar esta sequência na ordem:
1. Calcular Dano_bruto
   → Se ataque físico: ATQ_atributos + ATQ_arma × mod_raça + ATQ_equip × mod_raça
   → Se habilidade: (ATQ_total) × multiplicador_habilidade
   → Se mágico: ATQM_total × multiplicador_habilidade × mod_elemental

2. Verificar se é ataque normal (não-habilidade com acerto garantido):
   → Rola Esquiva_Perfeita do defensor
      → Se sucesso: MISS — fim da resolução
   → Calcula Chance_acerto = Precisão_A - Esquiva_B
      → Rola: se falha → MISS — fim da resolução

3. Verificar Crítico (apenas ataques físicos):
   → Rola CRIT do atacante
      → Se sucesso: Dano_bruto × 1,4

4. Aplicar DEF / DEFM:
   → Físico: Dano - DEF_leve, depois × (1 - DEF_pesada_redução%)
   → Mágico: Dano - DEFM_leve, depois × (1 - DEFM_pesada_redução%)

5. Aplicar resultado: subtrair do HP do defensor

6. Checar se HP ≤ 0 → KO

14. ADAPTAÇÕES ESPECÍFICAS PARA O GRID TÁTICO (8×16)
O LDI Tactics usa grid de 8 colunas × 16 linhas. As mecânicas acima precisam das seguintes adaptações:
Movimento por turno:
Casas_movimento = MOV_base_da_classe + bônus_equipamentos
(MOV é um atributo derivado, não primário — definido por classe)
Alcance de ataque:

Armas corpo a corpo: 1 casa adjacente (ortogonal + diagonal = 8 casas possíveis)
Armas médio alcance: 1–3 casas
Armas longa distância: 3–8 casas (verificar linha de visão)

Velocidade de ataque no grid:
No grid tático, ASPD não determina ataques por segundo — determina quantos ataques o personagem pode fazer no seu turno:
Ataques_por_turno = 1 (base para todos)
Se ASPD ≥ 175: 2 ataques por turno
Se ASPD ≥ 184: 3 ataques por turno
Isso significa que AGI e DES altos permitem múltiplos ataques no mesmo turno, mantendo a lógica de Ragnarok adaptada para turnos.
Conjuração no grid:
Habilidades com conjuração variável > 0 exigem que o personagem fique imóvel por N turnos antes de disparar:
Turnos_de_conjuração = ceil(Conjuração_total ÷ 1,0)
(Cada 1 segundo de conjuração = 1 turno parado)
Durante a conjuração, o personagem pode ser interrompido se receber dano acima de um threshold — a habilidade cancela e o SP é gasto mesmo assim.
Pós-conjuração no grid:
Depois de usar uma habilidade, o personagem fica sem poder usar outras habilidades por N turnos:
Turnos_de_delay = ceil(Pós_conjuração_final ÷ 1,0)

17. NOTAS PARA O AGENTE DE CÓDIGO

Todos os valores de atributo são inteiros. Cálculos intermediários podem ter decimal, mas o resultado final de dano é arredondado para baixo.
A ordem da sequência de resolução (seção 12) é rígida. Esquiva Perfeita é checada antes da esquiva normal. Crítico é aplicado antes da DEF.
Modificadores de raça/tamanho nunca afetam ATQ de atributos — apenas ATQ de arma e ATQ de equipamentos.
Bônus de ASPD em % não somam sobre valor exibido — calculam sobre o valor real com decimais.
Conjuração fixa e variável são calculadas separadamente e somadas no final — nunca calcular como uma coisa só.
Pós-conjuração ≠ Recarga. Pós-conjuração bloqueia tudo. Recarga bloqueia só aquela habilidade.
A tolerância a status negativos reduz a CHANCE de aplicação e a DURAÇÃO — são dois cálculos separados.
No grid tático, ASPD vira ataques/turno — não é medido em tempo real. Usar a tabela de conversão da seção 14.

SINERGIAS NOVAS COM OS 20
Time anti-habilidade:
Doutor Ferrugem suprime habilidade especial → Mãos de Osso ataca com Incisão que ignora metade da DEF → Língua de Cobra silencia o que sobrou → alvo não usa habilidade, não tem DEF, não pode contra-atacar.
Time de pressão de Sangramento:
Espinho Negro aplica Sangramento em área → Rugido dá +1 MOV para Relâmpago Branco → Relâmpago entra com 3 ataques em alvos já sangrando → Sangramento + dano direto elimina em 2 turnos algo que levaria 5.
O counter de Mãos de Osso:
Tocha Sagrada curando ele para manter o bônus de ×1,3 ativo indefinidamente é forte — mas Doutor Ferrugem suprime a habilidade de cura de Tocha por 1 turno. Um turno sem cura, Mãos de Osso cai para 25% de HP acidentalmente. Fica em ×1,6. Situação ficou ainda mais perigosa.
O time que controla cada sentido:
Espinho Negro aplica Cegueira. Língua de Cobra aplica Silêncio. Névoa da Manhã cria névoa que bloqueia mira. Doutor Ferrugem suprime habilidade. Um alvo sem visão, sem habilidades, sem mira e com a próxima habilidade travada... está morto antes de agir.

    { scale: 1.02 }, { scale: 0.99 }, { scale: 1.01 }, { scale: 1 },
  ],

export function screenShake(perfil, elementRef) {
  if (!elementRef?.current) return;
  const frames = SHAKE_PROFILES[perfil];
  if (!frames.length) return;

  let i = 0;
  const interval = 60; // ms por frame

  const tick = setInterval(() => {
    if (i >= frames.length) {
      clearInterval(tick);
      elementRef.current.style.transform = '';
      return;
    }
    const f = frames[i];
    if (f.scale) {
      elementRef.current.style.transform = `scale(${f.scale})`;
    } else {
      elementRef.current.style.transform = `translate(${f.x}px, ${f.y}px)`;
    }
    i++;
  }, interval);
}

    el.style.transition = 'background 0.5s ease, box-shadow 0.5s ease';
    // volta ao estado original via classe CSS
    setTimeout(() => {
      el.style.background = '';
      el.style.boxShadow  = '';
    }, 50);
  });
}
